# 领域模型与验收基线

## 1. 必须先固化的业务规则

在进入订单、支付、退款、库存、结算开发前，必须形成《业务规则与结算口径基线说明》。该文件至少覆盖：

- 用户下单规则
- 支付成功规则
- 订单取消规则
- 订单完成规则
- 售后申请规则
- 退款成功规则
- 库存扣减和回滚规则
- 履约任务生成规则
- 商户结算规则
- 加盟商结算规则
- 补偿和调整规则
- 对账和打款规则

## 2. 金额字段

金额统一用“分”为单位。

| 字段 | 含义 |
|---|---|
| `goodsAmount` | 商品金额 |
| `freightAmount` | 运费金额 |
| `discountAmount` | 优惠金额 |
| `prepaidAmount` | 福利卡抵扣金额 |
| `cashAmount` | 现金支付金额 |
| `payAmount` | 实付金额 |
| `refundAmount` | 退款金额 |
| `platformFeeAmount` | 平台服务费 |
| `franchiseAmount` | 加盟商应收或应结金额 |
| `merchantAmount` | 商户应付或应结金额 |
| `adjustAmount` | 调整金额 |
| `netAmount` | 最终净额 |

必须可验证：

- 订单应付 = 商品金额 + 运费金额 - 优惠金额
- 实付金额 = 福利卡抵扣金额 + 现金支付金额
- 可退金额大于等于本次退款金额
- 结算净额 = 正向账单金额 - 退款冲减金额 + 调整金额

## 3. 账单生成规则

- 支付成功：生成正向加盟商应收账单，生成正向商户应付账单。
- 退款成功：生成加盟商应收冲减账单，生成商户应付冲减账单。
- 补偿发生：生成补偿调整账单。
- 人工调整：必须填写原因、操作人、审批记录。
- 账单锁定：进入结算单后不允许直接修改原账单。
- 账单冲正：通过反向账单处理，不直接改历史金额。

## 4. 状态机清单

必须定义：

- 订单状态机
- 支付状态机
- 退款状态机
- 售后状态机
- 履约状态机
- 库存状态机
- 加盟商账单状态机
- 商户账单状态机
- 结算单状态机
- 打款状态机

每个状态机必须具备：

- 唯一编码
- 可读名称
- 允许流转目标
- 操作角色
- 是否需要审批
- 是否需要二次确认
- 状态变更审计

## 5. 幂等与唯一键

建议统一业务幂等键：

```text
bizType + bizSubType + bizOrderNo
```

适用场景：

- 支付成功回调
- 退款成功回调
- 福利卡余额变更
- 库存扣减
- 库存回滚
- 订单补偿
- 结算调整
- 人工修复

验收：

- 重复支付回调不能重复入账。
- 重复退款回调不能重复冲账。
- 福利卡余额变化必须有流水。
- 库存变化必须有流水。
- 结算单必须能追溯到订单、支付、退款和调整。

## 6. 线上支付与线下结算边界

新版 P0 的边界：

- 用户支付可以线上。
- 用户退款可以线上。
- 加盟商打款先保持线下确认。
- 商户打款先保持线下确认。
- 不把线上支付能力误写成线上自动分账能力。

结算状态建议：

- 待结算
- 已生成结算单
- 待确认打款
- 已线下打款
- 已完成
- 已冲正

## 7. 验收用例基线

P0 至少覆盖：

- 正常下单支付
- 福利卡和现金混合支付
- 订单取消
- 支付后退款
- 部分退款
- 全额退款
- 售后退款
- 库存扣减
- 库存回滚
- 商户账单生成
- 加盟商账单生成
- 退款冲减账单
- 结算单锁定
- 线下打款确认
- 重复支付回调
- 重复退款回调
- 人工调整
